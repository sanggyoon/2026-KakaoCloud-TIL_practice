## 3) Architecture

- 라우팅 테이블 vs 게이트 웨이 vs 로드밸런서

| 구분       | 라우팅 테이블    | 게이트웨이         | 로드밸런서                    |
| ---------- | ---------------- | ------------------ | ----------------------------- |
| **역할**   | 트래픽 방향 결정 | 네트워크 경계 통로 | 트래픽 분산                   |
| **레이어** | L3 (Network)     | L3 (Network)       | L4/L7 (Transport/Application) |
| **위치**   | 서브넷 연결      | VPC 레벨           | 서브넷 내부                   |
| **목적**   | 라우팅 규칙      | VPC 출입구         | 부하 분산, 고가용성           |
| **대상**   | IP 대역          | 전체 트래픽        | 특정 애플리케이션             |
|            |                  |                    |                               |

- 데이트 플로우

```bash
사용자 브라우저 (203.0.113.5)
      ↓
[인터넷]
      ↓
① Internet Gateway (igw-xxx)        ← VPC 진입
      ↓
② 라우팅 테이블 조회 (Public Subnet)
   0.0.0.0/0 → local 처리
      ↓
③ ALB (10.0.1.50)                  ← Public Subnet
   - 헬스체크
   - 타겟 그룹에서 EC2 선택
      ↓
④ 라우팅 테이블 조회 (Private Subnet)
   10.0.2.0/24 → local
      ↓
⑤ EC2-1 (10.0.2.10) 또는 EC2-2 (10.0.2.20)
```
